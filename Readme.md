Loose Mocks:
- Fewer lines of setup code
- Setup only what is relevant
- Default values
- Less brittle tests
- Existing tests continue to work

Strict Mocks:
- More lines of setup code
- May have to setup irrelevant things
- Have to setup each called method
- More brittle tests
- Existing tests may fail
